⚠️ このページについて

このページには意図的に重複したIDが含まれています。

HTML仕様では、1つのページ内で同じIDを複数回使用することは禁止されていますが、実際にはこのようなエラーが発生することがあります。

拡張機能はこのような重複IDを検出し、警告を表示します。

テスト項目

  • ✓ ユニークなID (#unique-id)
    期待: ✓ アンカー発見 (正常)
  • ⚠ 重複ID (#duplicate-id) - 1回目のリンク
    期待: ⚠ 重複ID検出 (Duplicate ID Detected)
  • ⚠ 重複ID (#duplicate-id) - 2回目のリンク
    期待: ⚠ 重複ID検出 (Duplicate ID Detected)

検証ポイント

  • ページ内に同じIDが複数存在する場合、重複ID警告が表示される
  • ステータスは「anchor-duplicate」として検出される
  • 重複IDは別カテゴリとして集計され、フィルタリング可能
  • ユニークなIDは正常として検出される

📍 Unique ID

これはユニークなIDです。ページ内で1回のみ使用されています。

id="unique-id"

⚠️ Duplicate ID - 1つ目

これは重複したIDです（1つ目）。

id="duplicate-id"

HTML仕様違反: 同じIDがページ内に複数存在します。

⚠️ Duplicate ID - 2つ目

これは重複したIDです（2つ目）。

id="duplicate-id"

HTML仕様違反: 同じIDがページ内に複数存在します。

テスト結果の確認方法

  1. 拡張機能のアイコンをクリックして起動
  2. 「Check Links」ボタンをクリック
  3. 結果を確認:
    • unique-id → ✓ アンカー発見（緑色）
    • duplicate-id（両方） → ⚠ 重複ID検出（黄色/オレンジ色）
  4. フィルターで「重複ID」を選択すると、重複IDのみが表示される
  5. 統計情報に重複ID数が表示される

💡 重複IDの影響

  • JavaScriptの document.getElementById() は最初の要素のみを返す
  • アンカーリンクのジャンプ先が不定になる
  • CSSのスタイリングが意図しない要素に適用される可能性
  • アクセシビリティツールが正しく動作しない可能性

→ 実際のWebサイトでは重複IDを修正することを強く推奨します
