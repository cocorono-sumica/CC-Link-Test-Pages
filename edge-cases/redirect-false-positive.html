⚠️ 重要: もしリダイレクトとして検出されたら、それは誤検知です。検出ロジックの改善が必要です。

🎯 関数定義内のlocationコード

以下のページは関数定義内にlocation.hrefを含みますが、即座に実行されるわけではありません。

  • 関数内にlocation.hrefを含むページ 正常であるべき
    コード例: 
    function doRedirect() {
    location.href = 'other-page.html';
}
    期待: ✅ 正常 (リダイレクトとして検出されない)
    理由: 関数定義内のコードは即座に実行されない
  • ASP.NET __doPostBack関数を含むページ 正常であるべき
    コード例: 
    function __doPostBack(target, arg) {
    document.forms[0].submit();
}
    期待: ✅ 正常 (フォーム送信として検出されない)
    理由: ASP.NETの標準関数定義

🎯 イベントハンドラ内のlocationコード

イベントハンドラ内のコードはユーザー操作によって実行されるため、自動リダイレクトではありません。

  • onclickでリダイレクトするボタンがあるページ 正常であるべき
    コード例: 
    <button onclick="location.href='other-page.html'">移動</button>
    期待: ✅ 正常 (リダイレクトとして検出されない)
    理由: ユーザークリックがトリガー
  • addEventListenerでリダイレクト処理を登録しているページ 正常であるべき
    コード例: 
    button.addEventListener('click', () => {
    location.href = 'other-page.html';
});
    期待: ✅ 正常 (リダイレクトとして検出されない)
    理由: イベント駆動のコード

🎯 コメント・文字列内のlocationコード

コメントや文字列内のコードは実行されません。

  • コメント内にlocation.hrefを含むページ 正常であるべき
    コード例: 
    // location.href = 'other-page.html';
/* window.location = 'example.com'; */
    期待: ✅ 正常 (リダイレクトとして検出されない)
    理由: コメントは実行されない
  • 文字列内にlocation.hrefを含むページ 正常であるべき
    コード例: 
    const example = "location.href = 'url'";
console.log('window.location = ...');
    期待: ✅ 正常 (リダイレクトとして検出されない)
    理由: 文字列リテラルは実行されない

🔍 検証ポイント

  • 関数定義の除外: function name() { location.href = ... } が除外されること
  • イベントハンドラの除外: onclick, addEventListener 内のコードが除外されること
  • コメントの除外: // や /* */ 内のコードが除外されること
  • 文字列の除外: "..." や '...' 内のコードが除外されること

⚠️ 誤検知が発生した場合

誤検知が発生した場合、以下の改善が必要です：

  • 関数定義パターンの正規表現を改善（ネストされた関数への対応）
  • コメント・文字列の除外ロジック追加
  • イベントハンドラ属性（onclick等）の除外強化
