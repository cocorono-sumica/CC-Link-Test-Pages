💡 内訳（ローカル開発環境）:
• 正常リンク: アンカーテスト(4) + 内部リンク(3) + 特殊ケース(4) = 11件
• エラー: 404ページ(1) + ソフトエラー×2(2) = 3件
• リダイレクト: Metaリダイレクト(1) ※200 OKだがリダイレクト情報を検出

⚠️ 注意: エラーページセクションのリンクをチェックすると、各エラータイプ（404、ソフトエラー、リダイレクト）の検出機能をテストできます。

🌐 公開ページ（GitHub Pages）での検出について:
公開ページ（https://cocorono-sumica.github.io/CC-Link-Test-Pages/）で拡張機能を実行すると、検出結果が異なる場合があります。

主な違い:
• エラー数: 4件程度（期待値より多い）
• リダイレクト数: 1-4件（環境により変動）

理由:
• GitHub Pagesでは、404.htmlなどのエラーページが実際のHTTPステータスコード（404など）を返すことがあり、その場合でもタイトルに「404」「エラー」などの文言が含まれるため、ソフトエラーとして誤検出される可能性があります
• 開発環境（Live Server）では全てのHTMLファイルがHTTP 200を返すため、ソフトエラー検出が正しく機能します
• この挙動は、検出ロジックの改善余地を示しています（詳細は下記「改善提案」を参照）

使い方

  1. 上記のカテゴリから、テストしたい項目のリンクをクリック
  2. テストページが開いたら、拡張機能のアイコンをクリック
  3. 「Check Links」ボタンをクリックしてリンクチェックを実行
  4. 結果を確認し、期待通りの動作をしているか検証

検証できる項目

  • アンカーリンク: 同一ページ内アンカー、別ページアンカー、エンコード処理、重複ID検出
  • 内部リンク: 正常なリンク、404エラー、相対パス/絶対パスの処理
  • 特殊ケース: 空のhref、非表示要素の検出、mailto:/tel:などの特殊プロトコル、各種リダイレクト
  • エラー検出: ソフトエラー（タイトル/コンテンツベース）、自動転送の検出
  • UI/UX: ハイライト機能、ジャンプ機能、エラー番号表示
