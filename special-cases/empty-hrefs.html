表示されている空のhref

UI上で表示されている空のhrefリンクです。これらはエラーとして検出されます。

  • ✗ 空のhref（表示）
    期待: Empty href (Visible) - エラー
  • ✗ 空のhref（表示）- 2つ目
    期待: Empty href (Visible) - エラー

非表示の空のhref

UI上で非表示の空のhrefリンクです。これらは保留として検出されます。

  • ⚠ 空のhref（非表示）
    期待: Empty href (Hidden) - 保留
  • ⚠ 空のhref（非表示）- 2つ目
    期待: Empty href (Hidden) - 保留

検証ポイント

  • 空のhref属性（href=""）が正しく検出される
  • 表示されている空のhrefは「エラー」として扱われる
  • 非表示の空のhrefは「保留」として扱われる
  • ステータスが「empty-visible」または「empty-hidden」になる
  • 専用のフィルターでそれぞれ表示できる

テスト結果の確認方法

  1. 拡張機能のアイコンをクリックして起動
  2. 「Check Links」ボタンをクリック
  3. 結果を確認:
    • 表示されている空のhrefは Empty href (Visible) でエラー表示
    • 非表示の空のhrefは Empty href (Hidden) で保留表示
  4. フィルター機能で「空欄（表示）」と「非表示空欄（保留）」をそれぞれ確認
  5. 統計情報で空欄数が正しくカウントされることを確認

💡 空のhrefについて

  • 空のhref（href=""）は現在のページを指します
  • クリックするとページがリロードされます
  • アクセシビリティの観点から推奨されません
  • JavaScript等で動的に設定される場合もありますが、基本的には避けるべきです
